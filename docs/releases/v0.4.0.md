# RiffNotes v0.4.0

This release captures the current Windows-focused build after the Google Drive account, audio output, Preferences, and Masters library iterations.

## Highlights

- Select and remember Windows playback output devices.
- Connect to a Google Drive account with bundled OAuth configuration.
- Handle browser OAuth callbacks more clearly, including visible/copyable errors.
- Browse Google Drive folders, create a RiffNotes folder, and remember the chosen remote sync root.
- Improve Preferences layout and disable/enable Google Drive actions based on connection state.
- Add copy buttons for activity/error messages.
- Remember whether the waveform/player panel is collapsed between app runs.
- Add a Masters library view so master/reference recordings can be opened and marked up with sections.
- Keep the existing waveform, note, section, boost, channel-processing, export, cache, and fingerprint-review workflows from v0.3.0.

## Verification

- `flutter test`
- `flutter build windows --no-pub`

## Notes

Direct Google Drive upload/download is still being built in slices. This version establishes account connection and remote folder selection, while local-folder sync remains available.

This is still experimental software. Windows is the primary development target. Android/Kindle Fire support remains planned around playback, note review/editing, sections, and sync.
